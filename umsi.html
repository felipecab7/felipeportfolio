UMSI Website


Target audience

The School of Information website targets prospective students, employers, community partners, alumni, donors and current students. In order to gain more knowledge on how to improve the usability of the site, I chose the most important target user --prospective students-- and conducted a Heuristic Evaluation analysis of the tasks they would carry out when exploring the site for the first time. As I stepped through each task, I recorded issues based on Nielsen’s 10 usability heuristics for user interface design, assigned a severity rating, and presented both written recommendations and redesigns that I believe can improve the learnability, efficiency, memorability and satisfaction of the site.



Methodology

In order to create a function/task list, I first identified the root needs prospective students have in the context of choosing a major. To do this I used the Jobs-to-Be-Done Framework --developed by pioneer Tony Ulwik back in 1991 and later popularized by Harvard Business School professor Clayton Christensen in 1991-- in order to identify the core functional, social, and emotional needs of prospective students.


User case 1: When I am deciding whether UMSI is the right fit for me, I want to know if UMSI and I share the same values and beliefs so I can feel connected and safe - feeling of belonging. Emotional need.


User case 2: When I am deciding whether UMSI is the right fit for me, I want to learn what jobs are in high demand so I can be certain about job security in the future. Emotional need.


User case 3: When I am deciding whether UMSI is the right fit for me, I want to learn how much money I can potentially make so I can achieve financial security. Emotional need.


User case 4: When I am deciding whether UMSI is the right fit for me, I want to quickly get a picture of the companies I would be working for so I can assess whether their values align with mine. Emotional need.


User case 5: When I am deciding whether UMSI is the right fit for me, I want to get information about programs offered so I can assess career outcomes. Functional need..



Needs Summary

UMSI site targets prospective students who want to:


1: Feel connected and safe feeling of belonging. Emotional need.


2: Know if UMSIs values are in sync with their personal values and beliefs. Emotional need..


3: Have job security in the future. Emotional need.


4: Achieve financial security. Emotional need.


5: Assess career outcomes. Functional need.


6: Explore possible workplace and industry opportunities. Functional need.


7: Know how to apply to UMSI. Functional need.


8: Contact UMSI faculty/staff for questions. Functional need.

Rating system

Keeping in mind Nielsen’s 10 usability heuristics for user interface design I recorded each problem and assigned one of the following severity ratings to it:


Minor Issue: Hinders the user's ability to navigate and should be fixed when possible


Major Issue: Frustrates or confuses users and requires repair as soon as possible


Catastrophic Issue: Prohibits users from performing their given task and requires an immediate modification


Design Recomendations

After laying out problems and possible solutions, I used Figma to redesign the content based on recommendations.



Findings and Recommendations


Problem 1


Actionable and non-actionable elements of the site have the same color. When hovering over the menu handle “About UMSI'', the color changes from green to yellow (the same yellow of the title “Information Changes Everything”, the button “UMSI” COVID-19 Updates and Resources” and “Header Menu”). Since the same yellow is assigned to multiple elements on the same page, it is difficult for the user to associate it with an indication of status. Internal consistency is also broken as the design of the title “information changes everything” looks almost the same as the other yellow elements.





Heuristic violated: Visibility of system status Severity: Minor Issue (Hinders the user's ability to navigate and should be fixed when possible)





Problem 2


Main buttons on the main menu and buttons on the header menu at the top of the page don’t show any feedback to the user when hovering over while drop-down menu buttons change their format to bold. This breaks internal consistency as the user should not have to use cognitive resources to understand why different menus give feedback and some don’t. When users are first exposed to a menu feedback, they build a mental model of how this feedback works and assigns a meaning to it. When exposed to the same cue (another button) and the feedback is not present, users might see it as an error and ignore it.





Heuristic violated: Consistency and Standards Severity: Minor Issue - Hinders the user's ability to navigate and should be fixed when possible


Recommendation: This can be fixed by giving all actionable buttons the same or similar feedback so users don’t have to rely on extra cognitive effort to understand their function or meaning.



Problem 3



Drop-down menus are too long and have too many options. This increases visual-search time (amount of time it takes the user to visually identify the piece of information they are looking for) and movement time (time it takes to move the cursor to get to the element). Fitts' law states that the amount of time required for a person to move a mouse cursor to a target area is a function of the distance to the target divided by the size of the target. In the screenshot below, we could see how only five menu options are within reach above the second green line. Past this line, options lose visibility.





Heuristic violated: Consistency and Standards. Severity: Major Issue - Frustrates or confused users and requires repair as soon as possible.


Recommendation: This problem could be fixed by having a mega menu or rectangular menu. This will prevent any scrolling problems that users might encounter when using a small screen and will reduce the time it takes to both visually identify the button and click on it.





Taking these into consideration, I reorganized the content and created a megamenu where the visual-search time and movement time are improved. I then placed the same green line and we can see how most of the menu options (9) are now within reach with only one exception.



Problem 4



Titles and subtitles on the main page are not properly aligned. The title "Information Changes everything" is too close to the top edge and is not aligned with other elements of the site such as the “Changing Access” title. This creates a sense of unease and imbalance.





Heuristic violated: Aesthetic and Minimalist Design . Severity: Minor Issue - Hinders the user's ability to navigate and should be fixed when possible


Recommendation: This problem could be fixed by having a mega menu or rectangular menu. This will prevent any scrolling problems that users might encounter when using a small screen and will reduce the time it takes to both visually identify the button and click on it.






Problem 5



The Header, Navigation Menu, Content and Footer sections appear to be grouped together due to the white background and different sections are too close to each other. This might confuse users as they can’t easily identify what the different sections of the site are. In the case of the UMSI homepage, the lack of a change in background color, makes sections like the header and content look like one piece of information --decreasing discoverability of relevant information. Additionally, the white space between the articles in the “News” section is the same as the “Events'' section. This small detail makes the “Events” section harder to identify.





Heuristic violated: Aesthetic and Minimalist Design . Severity: Minor Issue - Hinders the user's ability to navigate and should be fixed when possible


Recommendation: This problem could be fixed by having a mega menu or rectangular menu. This will prevent any scrolling problems that users might encounter when using a small screen and will reduce the time it takes to both visually identify the button and click on it.





Takeways


Two catastrophic, four major and two minor usability problems in the site’s navigation menu, homepage, mission page and BSI were uncovered.


Catastrophic Issues


Too many colors on the homepage: White, blue, purple, black, green. Heuristic: Aesthetic and Minimalist Design.


Picture at the top of the “Mission” page occupies most of the space. Heuristic: Aesthetic and Minimalist Design.


Major Issues


Drop-down menus are too long and have too many options. Heuristic: Consistency and Standards.


The Header, Navigation Menu, Content and Footer sections appear to be grouped together due to the white background and different sections are too close to each other. Heuristic: Aesthetic and Minimalist Design.


Sections of the page have a very similar visual weight. Heuristic: Aesthetic and Minimalist Design.


Users need to scroll down, scan the video, and read the text in order to find the most important information, increasing visual-search time. Heuristic: Recognition rather than recall.


Minor issues


Actionable and non-actionable elements of the site have the same color. Heuristic: Visibility of System Status

Note on Information Processing and Usability


The quicker and more ef effectively users are exposed to information that meets their needs, the higher the likelihood that they will make an effort to engage. This has to do with the way we process information --Daniel Kaneman in his book “Thinking Fast and Slow” describes the two modes in which we process information:.


System 1 (Fast): The automatic, intuitive and emotional part of the brain which operates with little or no conscious effort and no sense of voluntary control. We engage system 1 to make intuitive decisions without having to use much cognitive resources.


System 2 (Slow): The analytical and rational part of the brain which takes care of effortful mental activities that demand it, including complex computations and analyzing vast amounts of information. Slower system of the brain. Takes up most cognitive resources.


We make many of our daily decisions based on the suggestions System 1 gives System 2: if we see a stop sign (we stop immediately), if we feel anxious (we check social media), if we are in a room full of people (we follow stereotypes), if we encounter a new portfolio website that is aesthetically pleasing (we trust it). We have built-in neurological pathways that allow us to make sense of the world without having to use much of our cognitive resources. System 1 and System 2 exist to minimize effort so we can concentrate on demanding tasks that require our attention: building a new habit, engaging in a debate, reading a long text, solving a math problem. Following this logic, the question of whether a website has good usability (how easy and pleasant are user interfaces to use) or not, is a question of how much of system 1 and 2 is at play when a user experiences an interface interaction. System 1 helps us scan relevant information and judge credibility on the spot. If information on a website is inconsistent, irrelevant, does not match real world conventions and is not easily visible, users' ability to quickly build mental models will be hindered, causing them to rely on System 2 to make choices. In other words, the more we make people “think”, the more likely they are to go somewhere else where a high cognitive effort is not required. I mention these two systems throughout the analysis to explain the reasons behind each argument.
